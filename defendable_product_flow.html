03 · Product map

What each part owns today

Part 1 · ~85%
Analyse

defendable_analyser_v3

Disruption in → factual layout the lawyer can correct.

  • ICC / flight lookup intake
  • Editable LOF sector table
  • Chronological story + evidence status
  • Confirm LOF hard gate
  • Keyword draft classification (Zone 2)
Part 2 · Decide live
Decide

Trees + framework + G1

Locked facts → deterministic legal direction aligned to the Complete Legal Decision Framework.

  • Framework map (14 claim types · 20 DTs · 16 U-nodes · 6 verdicts)
  • Confirm LOF → type-map priority → DT gates
  • v2 thinking trail (E1…En) + CONDITIONS BEFORE FINAL RESPONSE
  • CRIT / IMPO evidence priorities · EVIDENCE_HOLD under CONDITIONS
  • G1 sign-off → editable Case Handoff Pack
  • Confirm → Manage case (summary, evidence index, legal position, packets)
Part 3 · handoff closed
Manage

261Claims platform

Case lifecycle after Confirm into Manage — engine facts already on the file.

  • Overview shows verdict, summary, locReady / Awaiting LOC
  • Evidence seeded from handoff marks + CRIT/IMPO
  • Docs in intake / legal_drafts / evidence_index (valid folders)
  • LOC paste/flag on case when claim letter arrives
04 · Lawyer gates

Where the system must stop

No automatic filing. Each gate needs a named lawyer action.

G0 · Confirm LOF Lock the factual record. Built now in Analyser.
G1 · Legal opinion Approve / override / send-back-for-evidence in Decide workspace. Built now — does not unfreeze G0.
G2 · Document send Approve holding / defence / settlement before send.
G3–G4 · Money / court Authorise settlement amount or escalate to ADR/court.
